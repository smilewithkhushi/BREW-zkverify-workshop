# ZKVerify Proof Submission & On-Chain Verification

---

## Overview

After generating the proof locally, we submit it to **ZKVerify** (a blockchain that verifies ZK proofs and aggregates them). Once aggregated, the proof is attested on **Sepolia** and we verify it on-chain via a smart contract.

```
Proof (local)  →  ZKVerify Volta  →  Aggregation  →  Sepolia Contract  →  true ✓
```

---

## Phase 1 — Submit Proof to ZKVerify

### Step 1: Setup the project

From inside `prep/proof-submission/`:

```
npm init -y && npm pkg set type=module
npm i zkverifyjs dotenv
```

### Step 2: Add your seed phrase

Create `.env.local`:
```
SEED_PHRASE = "your twelve word seed phrase here"
```

> Never commit this file. It is in `.gitignore`.

### Step 3: Run the submission script

```
node index.js
```

**What the script does:**
1. Reads binary `proof` and `vk` from `magic_square/target/`
2. Connects to ZKVerify's **Volta Testnet** using your seed phrase
3. Submits the proof as `UltrahonkVariant.Plain`
4. Subscribes to `NewAggregationReceipt` event
5. Saves `aggregation.json` once the proof is aggregated on domain `0` (Sepolia)

**Successful output:**
```
Included in block {
  blockHash: '0x...',
  proofType: 'ultrahonk',
  domainId: 0,
  aggregationId: 269036,
  statement: '0x84a7c73e...',
  ...
}
```

### Step 4: Check aggregation.json

Once the script completes, `aggregation.json` is saved:

```json
{
  "root": "0xb848677d...",
  "proof": [],
  "numberOfLeaves": 1,
  "leafIndex": 0,
  "leaf": "0x84a7c73e029e2822a0df98ed3b749db49957213f1b39aef383ed450048d8ce1d",
  "domainId": 0,
  "aggregationId": 269036
}
```

> Keep this file — you need these values to call the smart contract.

## What Does `checkHash` Actually Do?

```
_leaf  →  verifyProofAggregation(zkVerify proxy)
                    ↓
       Checks merkle inclusion in aggregation root
       posted by ZKVerify on Sepolia
                    ↓
              returns true ✓
```

The ZKVerify proxy holds aggregation roots posted from Volta. `verifyProofAggregation` checks that your `_leaf` (the statement hash of your proof) is included in the merkle tree for the given `aggregationId`. If it is — your proof is verified on-chain.
